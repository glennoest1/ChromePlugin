# Chrome Web Store Screenshot Plan

Use clean demo data only. Do not include real account names, tokens, customer content, or production URLs.

## Required

- 128x128 PNG icon: `bug-black-box/icons/icon128.png`
- At least one screenshot.

## Recommended Screenshot Set

1. Popup idle state with Current tab / All tabs recording choices.
2. Popup recording state after Start Recording.
3. Report preview showing counts, screenshot area, and export actions.
4. Session replay viewer with captured replay events.
5. Downloaded Markdown report opened in a viewer or editor.

## Capture Checklist

- Use `test-page.html` or another non-sensitive local test page.
- Confirm no passwords, tokens, cookies, API keys, or private URLs are visible.
- Prefer 1280x800 or 1600x900 screenshots for store review clarity.
- Keep browser UI minimal and avoid unrelated tabs.
- Retake screenshots after the final release ZIP is loaded unpacked.
